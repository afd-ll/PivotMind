# 玄枢架构重构计划 v2.0

> 基于自主/非自主两层神经架构的认知系统设计

---

## 一、核心认知：学习的本质

```
节点同时激活 → 边置信度上升（学习）
                      ↓
               持久化到记忆（存储）
                      ↓
         同类激活模式更容易走通（强化）
                      ↓
         另一条竞争路径激活更强（自然衰退）
```

**不是"等人说对才学"，而是"一起点火就焊"。**

---

## 二、两层神经架构

### 自主层（Autonomic）—— 运行时不需外部干预

| 机制 | 行为 | 对应代码 |
|------|------|----------|
| 节点同时激活 | 两个节点在同一次推理中被激活 → 它们之间的边的置信度上升 | 新建 `src/autonomic_learner.c` |
| 路径竞争 | 多次走通路径 A 不走路径 B → B 的置信度自然衰退 | 整合进联想推理循环 |
| 单字节点组合 | 不建词语拓扑。"学习" = 「学」节点 + 「习」节点同时激活 | 现有单字拓扑不变 |
| 激活扩散 | 从输入词出发，沿拓扑边传播激活能量，衰减衰减衰减 | 已有 `associative_reasoning.c` |

**关键词：运行时、自组织、无监督、持续在线。**

### 非自主层（Voluntary）—— 需要外部干预时触发

| 机制 | 行为 | 对应代码 |
|------|------|----------|
| 用户纠正 | 用户说"不对"→ 显式压下某条边的置信度 | 已有 `learn_from_dialog()` 的反馈处理 |
| 知识更新 | 用户主动教一条新知识 → 创建新节点+新边 | 已有 memory_store |
| 拓扑剪枝 | 置信度长期低于阈值的边 → 清理 | 已有 `cleanup_forgotten_knowledge()` |

**关键词：交互驱动、有监督、纠偏器、不上头。**

### 两层关系

```
自主层  ── 自然激活 ──→ 置信度涨落（神经递质式的缓慢调整）
                            ↑ 初始连接
非自主层 ── 用户纠正 ──→ 显式压/抬置信度（外部干预）
                            ↑ 错误路径
自主层  ── 竞争激活 ──→ 另一条路径更强，旧路径自然衰退
```

**非自主层不是定时器，是纠偏器。绝大多数时间自主层自己转。**

## 三、反馈系统的重新定位

现有 `learn_from_dialog()` 设计上有个根本偏差：**它用反馈来驱动学习**，把因果关系搞反了。

正确的设计：

```
学习 = 拓扑自己的事（同时激活 → 涨置信度）
反馈 = 外界说"你错了"（压置信度 / 重连）
记忆 = 持久化当前置信度状态
```

在代码上的体现：

```c
// ❌ 原来的思路：先等反馈再决定学不学
void learn_from_dialog(learner, input, response, feedback) {
    if (!feedback) return;  // 没有反馈就不学——不对！
}

// ✅ 新思路：自主层只管同时激活
void autonomic_activate(topology, input_concepts, response_concepts) {
    // 1. 激活输入概念节点
    // 2. 激活回复概念节点
    // 3. 同时激活的节点间涨置信度
    // 4. 竞争路径自然衰退
}

// ✅ 非自主层只做纠偏
void voluntary_correct(topology, wrong_edge_id) {
    // 1. 找到用户说"不对"的那条边
    // 2. 显式压下置信度
    // 3. 如果有更好的替代路径，提权
}
```

## 四、单字拓扑 → 词语的形成

**不做专门的词语拓扑。** 词语 = 多个单字节点同时激活。

```
"学习" 被输入
    ↓
「学」节点激活 (activation=0.7)
「习」节点激活 (activation=0.7)
    ↓
同时激活 → 「学」↔「习」边的置信度上升
    ↓
下次输入"学"，不仅激活「学」节点，
还会通过边自然激活「习」节点
    ↓
这种"一个字激活一串字"的效果 = 词语识别
```

优势：
- 没有词表边界，任何新组合自动成为"潜在词"
- 不用分词器，不用词典
- 对偶发组合（"草台班子"、"底层逻辑"）自然适应

## 五、实施计划

### Phase 1 — 自主学习器（基石）

**新文件：** `src/autonomic_learner.c`、`include/autonomic_learner.h`

实现核心机制：
1. **同时激活检测**：在一次推理中，记录被激活的节点对，涨置信度
2. **置信度涨落公式**：
   ```
   Δconfidence = learning_rate × (同时激活次数) × (1 - current_confidence)
   衰减: confidence *= decay_factor (每N次推理未同时激活)
   ```
3. **竞争衰退**：节点 A 有边到 B 和 C。B 连续被激活而 C 不 → C 的边衰退
4. **接入现有推理管线**：在联想推理完成后调用

**不引入新依赖，不改动现有代码结构。** `AutonomicLearner` 作为 `MasterTopology` 的一个新组件，在 `associative_reasoning` 完成后自动触发。

### Phase 2 — 改造 learn_from_dialog()

1. 分离出：
   - `autonomic_learn_from_dialog()` — 只做同时激活→涨置信度（不再等反馈）
   - `feedback_correct()` — 只做用户纠偏（原来的反馈逻辑）
2. 调用链：
   ```
   用户输入 → 推理 → 回复 → autonomic_learn_from_dialog(输入, 回复)
                                 ↓
                          用户可选纠正 → feedback_correct()
   ```

### Phase 3 — 废弃 reader 外部工具

reader 的共现建边逻辑直接内化到 `autonomic_learner` 中：
- 不再需要"先跑外部工具→导出种子→加载"的流程
- 读一本书 = 把书逐句推入推理管线 → 自主层自动建边
- 同理：QA 数据 = 把 QA 推入推理管线 → 自动建边

### Phase 4 — 实验验证

验证指标（不依赖 loss）：
1. **输入特定词 → 拓扑中相关节点被激活**
   - 例：输入"中国" → "北京"、"上海"、"首都" 等节点有较高激活值
2. **QA 学到的边可以表现为自然回复**
   - 例：喂过"什么是单片机"+"单片机是微型计算机" → 下次问"什么是单片机" → 相关概念被激活
3. **纠正后置信度真实下降**
   - 用户说"不对" → 那条边下次激活时强度变弱

## 七、认知调度中心

位于记忆系统和子拓扑之间，每轮对话计算意图向量，指导各子拓扑的搜索方向。

### 流程

```
用户输入
    ↓
compute_intent()
  └─ 上下文关联度 × context_bias
     └─ 新颖性因子 × novelty_bias
        └─ 效价偏好 × valence_bias
           └─ 连贯性奖励 × coherence_target
              → 归一化意图向量
    ↓
dialog_reason(..., intent_weights)
  └─ 每个子拓扑 × 对应权重
    ↓
do/while(retry 循环):
  ├─ self_verify_knowledge()
  ├─ dialog_generate() → response
  ├─ evaluate_draft()
  │   30%效价 + 40%连贯性 + 30%激活充足性
  └─ revise_and_retry()
       第1次: RETRY_FROM_POOL（候选池重排，不重搜）
       第2次: RETRY_WITH_SEARCH（缩域重搜）
       第3次: RETRY_FAILED（强制输出）
    ↓
cognitive_controller_snapshot()
  └─ 保存 prev_intent_weights + prev_satisfaction
```

### 三级降级策略

| 修正次数 | 策略 | 代价 |
|----------|------|------|
| 1 | 候选池重排（重新排序已有路径） | 极低，不触发新推理 |
| 2 | 缩域重搜（调整意图权重后重新推理） | 中，需要新推理 |
| 3 | 强制输出（已达上限） | 最低，直接输出已有结果 |

## 八、走边路径生成

### 问题

传统输出逻辑：激活传播 → 排序取 top-N 节点 → 顿号连接
→ 输出"人、工、智、能、是"，而不是"人工智能是"

网络中有 1.2M+ 条边，每条边编码了节点间的顺序关系和置信度。
这些边不应该只用于激活传播，也应该用于输出生成。

### 方案

从最高激活节点出发，沿边贪心走到下一个最优节点：

```
从起点出发:
  遍历每条出边，计算六维混合评分:
    base_score = 0.28×边权重 + 0.22×边置信 + 0.11×边动机
               + 0.28×目标激活 + 0.11×目标置信
    valence_modifier = 1.0 + 0.6 × 目标效价
    final_score = base_score × valence_modifier
  选最高分 → 走一步 → 标记已访问 → 继续
  直到无下一步或得分<0.05
```

### 混合评分设计

| 维度 | 类型 | 权重 | 说明 |
|------|------|------|------|
| 边逻辑强度 | 加法 | 0.28 | 基础连接强度 |
| 边置信度 | 加法 | 0.22 | 经过学习积累的置信度 |
| 边动机倾向 | 加法 | 0.11 | 用户偏好的动机偏置 |
| 目标节点激活 | 加法 | 0.28 | 当前推理中的激活值 |
| 目标节点置信 | 加法 | 0.11 | 节点自身稳定度 |
| 目标节点效价 | 乘法 | 0.6 | 乘法调节因子，-1→砍60%，+1→提升60% |

加法确保每个维度独立贡献，乘法让效价极端值时有一票否决能力。
